## 二、安装项目

### 问题与解决方案

#### 问题：系统保护的 Python 环境

直接运行 `pip install -e .` 遇到错误：

```bash
pip install -e .
```

错误输出:
```
× This environment is externally managed
├─> To install Python packages system-wide, try apt install
|   python3-xyz, where xyz is the package you are trying to
|   install.
|
|   If you wish to install a non-Debian-packaged Python package,
|   create a virtual environment using python3 -m venv path/to/venv.
```

#### 解决方案：使用虚拟环境

```bash
# 创建虚拟环境
python3 -m venv /opt/github/ai-ops-cil/venv

# 激活虚拟环境并安装项目
source /opt/github/ai-ops-cil/venv/bin/activate && pip install -e .
```

## 三、配置 Shell Alias

### 添加 Alias 到 ~/.bashrc

```bash
echo "alias @ai='ai-ops'" >> ~/.bashrc
```

### 验证 Alias 已添加

```bash
tail -5 ~/.bashrc
```

输出显示:
```bash
alias @ai='ai-ops'
```

### 使 Alias 生效

```bash
source ~/.bashrc
```

> 注意：每次打开新终端时，alias 会自动加载。

## 四、首次运行与配置初始化

### 首次运行触发配置文件创建

```bash
/opt/github/ai-ops-cil/venv/bin/ai-ops --help
```

输出:
```
未找到配置文件。已在 /root/.aiops/config.toml 创建默认配置，请填入 api_key 后重试。
```

### 查看生成的配置文件

```bash
cat ~/.aiops/config.toml
```

内容:
```toml
[llm]
base_url = "https://api.openai.com/v1"
api_key = ""
model = "gpt-4o"
temperature = 0.0
max_tokens = 256
timeout_seconds = 30

[safety]
comment_mode = true
extra_deny_patterns = []

[display]
language = "zh-CN"
animation_style = "spinner"
```

### 重要：配置 API Key

**必须设置 API Key 才能使用 CLI**。编辑配置文件：

```bash
nano ~/.aiops/config.toml
```

将 `api_key = ""` 改为您的实际 Key，例如：

#### 使用 OpenAI 官方 API
```toml
api_key = "sk-your-actual-openai-api-key"
```

#### 使用国内中转服务
```toml
base_url = "https://api.your-proxy.com/v1"
api_key = "sk-your-proxy-api-key"
```

#### 使用本地 Ollama
```toml
base_url = "http://localhost:11434/v1"
api_key = "ollama"  # Ollama 不需要真实 key
model = "qwen2.5-coder:7b"
```

## 七、常见问题

### Q1: 为什么不直接使用 pip install -e .？

**原因**: Debian/Ubuntu 系统默认使用 PEP 668 保护 Python 环境，防止系统包被 pip 修改。

**解决**: 使用虚拟环境或传递 `--break-system-packages` 参数（不推荐）。

### Q2: Alias 不生效怎么办？

**解决**:
1. 检查是否添加成功: `grep "@ai" ~/.bashrc`
2. 重新加载: `source ~/.bashrc`
3. 如果使用 zsh，需要编辑 `~/.zshrc` 而非 `~/.bashrc`

### Q3: 如何切换 Python 环境？

如果在不同 Python 版本之间切换：

```bash
# 删除旧虚拟环境
rm -rf /opt/github/ai-ops-cil/venv

# 使用新 Python 版本创建
python3.11 -m venv /opt/github/ai-ops-cil/venv
python3.11 -m venv /opt/github/ai-ops-cil/venv/bin/activate
pip install -e .
```

### Q4: 卸载项目

```bash
# 激活虚拟环境
source /opt/github/ai-ops-cil/venv/bin/activate

# 卸载
pip uninstall ai-ops

# 删除虚拟环境
deactivate
rm -rf /opt/github/ai-ops-cil/venv

# 删除配置文件（可选）
rm -rf ~/.aiops

# 删除 alias（可选）
sed -i '/alias @ai=/d' ~/.bashrc
```
